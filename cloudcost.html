CloudCostTracker: Automated Cost Monitoring

An enterprise-grade, serverless cost-management solution built on AWS. This system automates billing observability, threshold alerting, and executive reporting to ensure full transparency of cloud expenditures while maintaining a zero-trust security posture.

Project Objective

To implement an automated pipeline that monitors AWS "Estimated Charges," triggers instant alerts via SNS upon threshold breaches, and generates a weekly service-by-service cost dashboard hosted securely behind CloudFront Origin Access Control (OAC).

Architecture Overview

CloudWatch (Metrics) → SNS (Alerting) → EventBridge (Cron) → Lambda (Reporting) → S3 + CloudFront (OAC)
  • Monitoring & Alerting: CloudWatch Alarms track billing metrics and trigger SNS notifications for immediate budget awareness.
  • Automated Compute: AWS Lambda (Python 3.12) queries the Cost Explorer API to aggregate usage data and generate HTML visualizations.
  • Secure Distribution: Reports are served via Amazon CloudFront using Origin Access Control (OAC) to keep the S3 origin strictly private.
  • Data Visualization: Frontend integration using Chart.js to provide executive-level insights into service-specific cost drivers.

Security & Operational Excellence

  • Zero-Public S3 Origin: Applied "Block Public Access" to the S3 bucket, utilizing CloudFront OAC for secure, authenticated content delivery.
  • IAM Scoping: Assigned a least-privilege execution role to Lambda, strictly limiting permissions to ce:GetCostAndUsage and s3:PutObject.
  • Automated Lifecycle: Zero-touch reporting scheduled via EventBridge, reducing manual operational overhead.
  • Region-Aware Monitoring: Strategically configured metrics in us-east-1 to capture global account-wide spending data.
