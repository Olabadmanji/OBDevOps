Highly Available 3-Tier Architecture on AWS

A production-ready, highly available, and secure 3-tier architecture deployed on AWS. This system isolates the presentation, application, and database layers across multiple Availability Zones to ensure fault tolerance and security.

Project Objective

To design and implement a scalable enterprise network architecture that protects sensitive data in private subnets while delivering high-performance web traffic globally.

Architecture Overview

User → Amazon CloudFront → AWS WAF → Application Load Balancer (Public) → Auto Scaling EC2 (Private App) → Amazon RDS (Private DB)
  • Presentation Layer: Application Load Balancer distributing traffic across Availability Zones.
  • Application Layer: Auto Scaling Group of EC2 instances in private subnets, completely hidden from the internet.
  • Data Layer: Multi-AZ Amazon RDS database locked down in isolated subnets.

AWS Services & Security Additions

  • Amazon CloudFront & WAF: Global edge caching and protection against SQL injection.
  • Systems Manager (SSM): Secure, encrypted retrieval of database credentials.
  • VPC & NAT Gateway: Strict network routing allowing outbound internet access for private servers without inbound exposure.
  • Security Groups: Least-privilege firewalls restricting traffic tier-by-tier.
