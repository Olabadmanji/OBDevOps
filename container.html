Container Workload on AWS ECS Fargate

A containerized environment deploying a Node.js application using serverless container orchestration. This project leverages an Infrastructure as Code (IaC) approach to deliver a highly available, auto-scaling system with zero server management overhead.

Project Objective

To design and provision a secure, scalable container runtime using Terraform, ensuring that application workloads run in isolated private subnets with automated traffic routing via a Load Balancer and secure runtime credential injection.

Architecture Flow

Terraform (IaC) → Amazon ECR (Images) → Application Load Balancer (Traffic) → ECS Fargate (Private Compute) → Secrets Manager (Security)
  • Infrastructure as Code: Entire environment including VPC, IAM roles, and ECS clusters provisioned using Terraform for version-controlled infrastructure.
  • Serverless Compute: AWS ECS Fargate handles the container lifecycle, removing the need to manage or patch underlying EC2 instances.
  • Network Isolation: Deployed tasks in private subnets with no public IP addresses, utilizing NAT Gateways for secure outbound image pulls.
  • Traffic Orchestration: Application Load Balancer (ALB) performs health checks and distributes traffic across dynamically scaling container tasks.

Technical Highlights & Security

  • Dynamic Scaling: Integrated Target Tracking Auto Scaling to adjust container counts automatically based on real-time CPU and Memory utilization.
  • Secrets Management: Utilized AWS Secrets Manager to inject environment-sensitive variables directly into containers at runtime, avoiding hardcoded credentials.
  • Security Hardening: Implemented separate IAM execution and task roles to strictly follow the principle of least privilege.
  • Observability: CloudWatch Logs integration provides centralized log aggregation for real-time application monitoring and troubleshooting.
