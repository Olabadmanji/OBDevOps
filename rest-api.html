Serverless CRUD API with JWT Security

A secure, event-driven backend system built on AWS that handles enterprise-grade user authentication and high-performance CRUD operations. This architecture eliminates server management while ensuring 100% user data isolation.

Project Objective

To design a scalable, serverless backend that utilizes JSON Web Tokens (JWT) for stateless authentication, ensuring that data processing only occurs for verified requests while maintaining a "Scale-to-Zero" cost model.

Architecture Flow

Amazon Cognito (IDP) → API Gateway (JWT Validation) → AWS Lambda (Python Logic) → Amazon DynamoDB (NoSQL)
  • Identity Management: Amazon Cognito handles user sign-up, sign-in, and cryptographically signed JWT issuance.
  • Secure Routing: API Gateway acts as the entry point, rejecting unauthenticated traffic (401) before reaching compute layers.
  • Serverless Compute: Modular Python 3.12 functions process individual CRUD operations with scoped IAM permissions.
  • NoSQL Persistence: DynamoDB provides sub-millisecond latency for task storage using partitioned user IDs.

Technical Highlights & Security

  • Stateless Auth: Integrated a native JWT authorizer in API Gateway to offload authentication logic from the application.
  • Data Integrity: Implemented boolean-logic validation to ensure task completion statuses are stored as accurate data types.
  • Operational Monitoring: CloudWatch integration provides granular visibility into Lambda execution duration and error rates.
  • Least Privilege: Custom IAM execution roles ensure each function has only the minimum required access to DynamoDB tables.
